i finally got my hands on a mylo com-2! it's a cute little pocket-sized web browser and media player device from the pre-iphone era. because its primary focus is just browsing the web, running 3rd party programs isn't really something it's too concerned with, but it is able to run custom widgets much like android can. i love developing for weird, old, forgotten platforms, so let's try to make one!

of course i had to give it a gir charm.

back in 2008, sony had a "mylo labs" website where you could access tools and info for developing your own widgets, but of course, all of that precious information has been lost to time by now. by some stroke of luck, i was able to find a single archived thread on some old mylo forum where someone had uploaded a sudoku widget they developed! i think this is the only mylo widget file left on the web anywhere, so let's reverse engineer it!

deconstructing the sudoku widget

let's take a closer look at that sudoku.mylow widget. conveniently, mylow files are just renamed zip files, so we can easily look inside by renaming it to a .zip and extracting it.

seems like it's literally just a webpage with some xml files for metadata - extra convenient! let's see what's in those xml files:

"myloConfig.xml" is pretty barebones. maybe this is where optional advanced functionality, such as notifications or background updates, would be invoked? unfortunately i won't be able to find out unless i can get access to a more complex widget to dissect.

<?xml version="1.0" encoding="utf-8"?>
<!-- Copyright 2008 onshoresystems.com -->
<config xmlns="http://onshoresystems.com/mylo" version="1.0">
</config>

"widgetPackage.xml" is where all the important widget configuration and metadata is defined.

<?xml version="1.0" encoding="utf-8" ?>
<!-- Copyright 2008 OnShoreSystems.com -->
<widgetPackage xmlns="http://onshoresystems.com/mylo" version="1.0">
  <info>
    <packageName>iPhone Suduko Widget</packageName>
    <author>Apple</author>
    <abstract>Mylo page iPhone sample app</abstract>
    <version>1.000</version>
    <locale>US</locale>
    <engine>1.000</engine>
    <updateURL>http://www.onshoresystems.com/mylo/update/sudoku</updateURL>
    <siteURL>http://www.onshoresystems.com/mylo</siteURL>
    <minWidth>15</minWidth>
    <maxWidth>50</maxWidth>
    <defWidth>32</defWidth>
    <defHeight>32</defHeight>
    <createDate>2008-02-09</createDate>
  </info>
</widgetPackage>

this all seems pretty self-explanatory and easy to edit, so i might as well try to make my own widget!

creating a new widget

i might as well start with a classic "hello world"-style widget as a proof of concept. to do this, i'll create copies of the sudoku widget's index.html, myloConfig.xml, and widgetPackage.xml files and modify them as needed.

the index.html file i wrote just displays the text "hello mylo!" on a blue background.

<html>
	<head>
		<meta http-equiv="content-type" content="text/html; charset=utf-8">
		<meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=no">
		<title>hello</title>
	</head>
	
	<body style="background-color: #0000FF;">
		<p style="text-align: center; color: #FFFFFF;">hello mylo!</p>
	</body>
</html>

myloConfig.xml remained mostly unchanged, except for swapping out the xmlns for my own website.

<?xml version="1.0" encoding="utf-8"?>
<config xmlns="http://cyberdragon.digital/" version="1.0">
</config>

widgetPackage.xml was modified quite a bit to change the widget name, description, and author information.

<?xml version="1.0" encoding="utf-8" ?>
<widgetPackage xmlns="http://cyberdragon.digital/" version="1.0">
	<info>
		<packageName>Hello Mylo</packageName>
		<author>Frost Sheridan</author>
		<abstract>My first mylo widget!</abstract>
		<version>1.000</version>
		<locale>US</locale>
		<engine>1.000</engine>
		<updateURL>http://cyberdragon.digital/</updateURL>
		<siteURL>http://cyberdragon.digital/</siteURL>
		<minWidth>15</minWidth>
		<maxWidth>50</maxWidth>
		<defWidth>32</defWidth>
		<defHeight>32</defHeight>
		<createDate>2022-08-15</createDate>
	</info>
</widgetPackage>

i zipped up those three files into a "hello.zip" file, renamed it to "hello.mylow", and copied it over to the mylo. the widget installer accepted the widget file and displayed the metadata i entered. seem promising so far!

now all i need to do is add the widget in Widget Setting Mode and see what happens...

success!!! the widget showed up just as expected! it looks like the width and height units in widgetPackage.xml are a lot bigger than pixels - the 32 units specified seems to cover about half of the screen's width, so maybe 64 units corresponds to the entire width of the screen? i'll do some more experimentation with the xml file and see what else i can discover in the future.

this is super exciting though! i'm glad that creating a simple widget ended up being this simple to figure out.
